# AudioFractal вҖ” Real-Time Audio-Driven Fractal Visualization

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Dispatch 5 subagents in parallel, one per module group. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Build a complete real-time audio-driven fractal art visualization web app using custom GLSL fragment shaders (Mandelbrot, Julia, Burning Ship, High-order Mandelbrot), Three.js for WebGL management, and Web Audio API for sound analysis.

**Architecture:** Audio data flows: AudioEngine вҶ’ AudioAnalyzer (FFT/waveform) вҶ’ AudioFeatures (beat/RMS/centroid) вҶ’ FractalParameters (maps audio to shader uniforms) вҶ’ FractalMaterial (compiles GLSL) вҶ’ FractalRenderer (Three.js rAF loop). A ControlPanel UI sidebar lets users adjust mapping strengths, colors, themes, and fractal type.

**Tech Stack:** Vite (build tool), Three.js ^0.170.0, custom GLSL Fragment Shaders, Web Audio API (AnalyserNode, FFT=2048), vanilla JS ES Modules, CSS3 custom properties.

## Global Constraints

- Vanilla JavaScript ES Modules only вҖ” no TypeScript
- Three.js ^0.170.0, Vite ^6.x
- FFT size: 2048
- 4 fractal types: Mandelbrot, Julia, Burning Ship, High-order Mandelbrot
- 4 themes: Cyberpunk, Deep Ocean, Aurora, Inferno
- Responsive: desktop >1024px (sidebar 320px), tablet 768-1024px (sidebar 280px), mobile <768px (bottom drawer, fullscreen canvas)
- Vite base: './' for GitHub Pages relative paths
- package.json: `"homepage": "https://<user>.github.io/AudioFractal"`
- Keyboard shortcuts: Space=toggle audio, F=fullscreen, 1-4=switch fractal type
- Shaders: 80-150 lines each + common.glsl for shared utilities
- JSDoc on all public methods
- Tests: AudioFeatures.test.js and FractalParameters.test.js

## AudioвҶ’Fractal Parameter Mapping (core creative mapping)

| Audio Feature | Target Uniform | Mapping Function | Range |
|---|---|---|---|
| `lowBand` | `u_zoom` | zoom = 1.0 + lowBand * 3.0 * mappingStrength | zoom: 1.0вҖ“4.0 |
| `midBand` | `u_colorShift` | colorShift = midBand * 0.5 * mappingStrength | 0вҖ“0.5 |
| `highBand` | `u_maxIter` | maxIter = 50 + highBand * 450 * mappingStrength | 50вҖ“500 |
| `rms` | `u_brightness` | brightness = 0.5 + rms * 1.5 * mappingStrength | 0.5вҖ“2.0 |
| `beat` | `u_juliaC` | On beat: random perturbation to juliaC (Вұ0.1) | - |
| `spectralCentroid` | `u_saturation` | saturation = 0.3 + centroid * 1.4 * mappingStrength | 0.3вҖ“1.7 |
| `spectralFlux` | `u_distortion` | distortion = flux * 0.1 * mappingStrength | 0вҖ“0.1 |
| `zeroCrossingRate` | `u_juliaC[1]` (imag) | juliaC.imag = baseImag + zcr * 0.5 * mappingStrength | baseВұ0.5 |

Each mapping has a user-adjustable `mappingStrength` (0вҖ“1) controlled by sliders.

### AudioFeatures

```js
class AudioFeatures {
  constructor(config = { historySize: 43, beatThreshold: 1.3 })
  update(analyzer: AudioAnalyzer): void              // Call each frame
  getRMS(): number                                   // 0-1
  detectBeat(): boolean                              // true on beat frame
  getSpectralCentroid(): number                      // 0-1 normalized
  getLowFreqRatio(): number                          // 0-1
  getSpectralFlux(): number                          // 0-1 normalized
  getZeroCrossingRate(): number                      // 0-1
  getAllFeatures(): AudioFeatureSnapshot
}

// Beat detection algorithm:
// 1. Compute instant energy = sum of squared low-band bins
// 2. Compute average energy over history buffer
// 3. Beat when instant > average * beatThreshold
// 4. Store energy in circular history buffer
// 5. Cooldown: ignore beats within ~20 frames of each other (вүҲ430ms at 60fps)

// Spectral flux:
// 1. Store previous frame's spectrum
// 2. Flux = sum of positive differences between current and previous
// 3. Normalize by bin count

// Zero crossing rate:
// 1. Count sign changes in time-domain waveform
// 2. Divide by buffer length - 1
```

### Task 1.1: common.glsl (~50 lines)

Functions to implement:
- `vec3 hsv2rgb(vec3 c)` вҖ” standard HSVвҶ’RGB using the `K = vec4(1.0, 2.0/3.0, 1.0/3.0, 3.0)` method
- `vec3 palette(float t, vec3 a, vec3 b, vec3 c, vec3 d)` вҖ” `a + b*cos(6.28318*(c*t + d))` (ГҚГұigo QuГӯlez cosine palette)
- `float smoothColor(float iter, float zr, float zi, float maxIter)` вҖ” smooth iteration count: `iter + 1.0 - log2(log2(|z|))` for continuous coloring
- `float hash(vec2 p)` вҖ” simple hash: `fract(sin(dot(p, vec2(127.1,311.7)))*43758.5453)`
- `float noise(vec2 p)` вҖ” 2D value noise using floor/fract interpolation
- Helper: `float hueFromRGB(vec3 c)`, `float satFromRGB(vec3 c)`, `float valFromRGB(vec3 c)` вҖ” for hue rotation in shaders

Use `#ifndef COMMON_GLSL` / `#define COMMON_GLSL` / `#endif` guard.

### Task 1.2: mandelbrot.frag (~100 lines)

**Uniforms:** `u_resolution`, `u_center`, `u_zoom`, `u_maxIter`, `u_colorOffset`, `u_colorShift`, `u_hueShift`, `u_saturation`, `u_brightness`, `u_contrast`, `u_paletteA/B/C/D`, `u_time`

**Algorithm:**
1. Map `vUv` to complex plane coordinates: `cr = (uv.x - 0.5) * 3.5 / zoom * aspect + center.x`, `ci = (uv.y - 0.5) * 2.0 / zoom + center.y`
2. Run Mandelbrot iteration: `z = zВІ + c` where `c = cr + ci*i`, `zвӮҖ = 0`
3. Break when `|z|ВІ > 4.0` or `i >= maxIter`
4. Apply smoothColor for continuous coloring
5. Apply palette with color offset/shift
6. Apply HSV adjustments (hue shift, saturation, brightness, contrast)
7. Points inside set (iter == maxIter-1) вҶ’ black

### Task 1.3: julia.frag (~100 lines)

Same uniform structure as mandelbrot plus `u_juliaC` (vec2).

**Algorithm:**
1. Map UV to complex plane (same as Mandelbrot but with Julia-specific default zoom/center)
2. Run Julia iteration: `z = zВІ + c` where `c = u_juliaC` (fixed), `zвӮҖ = pixel_coord`
3. Same smoothColor and coloring pipeline

### Task 1.4: burningship.frag (~90 lines)

Same uniforms as mandelbrot.

**Algorithm:**
1. Map UV to complex plane (Burning Ship has different interesting regions: center at ~(-1.75, -0.03))
2. Run Burning Ship iteration: `zr = |zr|`, `zi = |zi|` before squaring, then `z = (|zr| + i*|zi|)ВІ + c`
3. Equivalently: `zr_new = zrВІ - ziВІ + cr`, `zi_new = 2*|zr*zi| + ci`
4. Same coloring pipeline

### Task 1.5: mandelbrot_alt.frag (~100 lines)

Same uniforms as mandelbrot plus `u_power` (float, default 3.0).

**Algorithm:**
1. Map UV to complex plane
2. Run generalized Mandelbrot: `z = z^n + c` where n = u_power
3. For power n, use De Moivre: `z^n = |z|^n * (cos(n*Оё) + i*sin(n*Оё))` where Оё = atan2(zi, zr)
4. Same coloring pipeline
5. Use `atan`, `pow`, `cos`, `sin` for polar form exponentiation

```js
// Key implementation details:
//
// Beat Detection (spectral energy method):
// - Maintain circular buffer of 43 frames (~0.7s at 60fps) of low-band energy
// - Each frame: energy = sum of squared magnitudes in low-band bins
// - Average energy = mean of history buffer
// - Beat when energy > average * threshold (default 1.3)
// - Cooldown: 20 frames minimum between beats
//
// RMS (Root Mean Square):
// - RMS = sqrt(mean of squared time-domain samples after centering around 128)
//
// Spectral Centroid:
// - centroid = sum(binIndex * magnitude) / sum(magnitude)
// - Normalize to 0-1 by dividing by max bin index
//
// Low Frequency Ratio:
// - lowRatio = lowBandEnergy / totalEnergy (all bins)
//
// Spectral Flux:
// - Store previous frame's spectrum
// - flux = sum(max(0, current[i] - previous[i])) / numBins
// - Normalize by dividing by 255
//
// Zero Crossing Rate:
// - Count sign changes in time-domain waveform (samples cross the 128 midpoint)
// - zcr = crossings / (bufferLength - 1)
```
